# 2026-05-06 KOMODO 聊天页 UI 重构 + record 包修复

## 修复：record Linux 编译错误
- **错误**：`record_linux 0.7.2` 缺少 `startStream` 方法，与 `record_platform_interface 1.5.0` 不兼容，导致 Linux 构建失败
- **修复**：`pubspec.yaml` 中 `record` 从 `^5.1.2` 升级到 `^6.2.0`
  - `record 6.2.0` 配套 `record_linux 1.3.0`，API 完全兼容
  - `start()` 签名变为 `start(RecordConfig config, {required String path})`
- **额外修复**：`chat_page.dart` 中 `colorScheme.colorScheme.brightness` → `colorScheme.brightness`

## 优化：聊天详情页 UI 重构（chat_page.dart）
- **移除**：用户信息卡片（`_buildPeerInfoCard`）、动态展示区（`_buildMomentsSection`）、快捷表情回复栏（`_buildQuickReplies`）
- **新的底部输入行**：`[🎤 录音按钮] [输入框] [😊 表情按钮] [+ 圆形加号按钮]`
- **可展开图标栏**：默认隐藏，点击 + 按钮展开，包含：图片 / 拍照 / 礼物 / 位置 / 更多
- **表情面板**：点击表情按钮展开/收起，点击表情直接发送
- **状态变量**：`_showIconBar`（控制图标栏展开）、`_showEmojiPicker`（控制表情面板）
- `+` 按钮为圆形（`BoxShape.circle`），展开时变为 `✕` 图标并高亮

## 优化：录音交互重构（chat_page.dart）
- **交互流程**：
  1. 点击左侧话筒图标 → 输入框切换为"长按录音"按钮条（_RecordState.ready）
  2. 长按录音条 → 弹出 `showModalBottomSheet` 录音浮层，显示脉冲话筒 + 实时波形动画 + 计时器（_RecordState.recording）
  3. 松手停止录音 → 浮层切换为预览模式，含播放按钮 + 进度条（_RecordState.preview）
  4. 点击播放 → Mock 播放，进度条动画推进
  5. 点击"发送"→ 添加语音消息到列表、关闭浮层；点击"取消"→ 丢弃录音、关闭浮层
- **消息内播放**：语音气泡点击播放/暂停（Mock），气泡宽度按时长动态缩放，播放中显示动态波形
- **状态机**：`_RecordState` enum（idle/ready/recording/preview）
- **浮层设计**：`_RecordingOverlay` 独立 StatefulWidget，通过 `StreamBuilder` + 轮询流订阅父状态变化
- **重要 Bug 修复（Overlay 替代 showModalBottomSheet）**：
  - `showModalBottomSheet` 打开新路由后，原 `GestureDetector`（`onLongPressStart/End`）的手势上下文被阻断，`onLongPressEnd` 永远不触发
  - **正确方案**：改用 `OverlayEntry`（`Overlay.of(context).insert()`）直接插入到页面层级
  - 手势全程在 `_buildVoiceRecordBar` 的 `GestureDetector` 子树内完成，松手正常触发 `_stopRecording()`
  - 浮层状态通过 `_overlayEntry.markNeedsBuild()` 驱动重建，在 recording ↔ preview 两个 UI 间切换
- **播放接入 just_audio**：预览和消息内播放均用 `just_audio` 真实播放（`setFilePath` + `play/pause`）
- **文件校验**：`stopRecording()` 后检查 `File(_recordedPath!).exists()` 和 `length()`，小于 100B 视为异常，输出 debugPrint 日志
- **_ChatMessage 新增 voicePath**：发送录音时把真实路径存入消息对象，支持消息气泡点击重播
- **取消删除临时文件**：`cancelRecording()` 调用 `File(path).deleteSync()` 清理
- `flutter analyze` 零 issue
