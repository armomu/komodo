### 未读数修复 — 加 is_read 字段（16:57）
- 问题：刷新列表时未读数 = 该 peer 全部历史消息数（因为 getUnreadCountByPeerId 只过滤 is_me=0，没有区分已读/未读）
- 修复：
  - `chat_database.dart`：messages 表加 `is_read INTEGER NOT NULL DEFAULT 0`，DB version 2→3，onUpgrade 用 ALTER TABLE 加列
  - `getUnreadCountByPeerId`：WHERE 条件加 `AND is_read = 0`
  - `clearUnreadByPeerId`：真正执行 `UPDATE messages SET is_read = 1 WHERE peer_id = ? AND is_me = 0 AND is_read = 0`
  - `insertMessage`：INSERT 时加 `is_read: 0`
  - `consumer_list_controller.dart`：`openChat()` 进入聊天时调用 `ChatDatabase.to.clearUnreadByPeerId(item.id)`
