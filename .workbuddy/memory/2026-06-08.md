# 2026-06-08 工作日志

## 音乐模块缓存功能实现

### 修改内容

1. **music_models.dart**
   - `PlaylistItem` 类增加 `String? cachedPath` 字段，记录网络歌曲缓存到本地的路径
   - 移除 `const` 构造器，改用普通构造器以支持 `cachedPath` 运行时修改
   - `localPlaylist` 从 `const List` 改为 `final List`，允许列表内元素的字段修改

2. **music_cache_service.dart**（新增）
   - `MusicCacheService` 静态工具类，管理音乐缓存生命周期
   - 缓存目录：`{应用文档目录}/music_cache/`
   - 基于 URL 的 base64Url 编码生成唯一文件名，保留原始扩展名
   - 核心方法：`getCacheDirectory()`, `getCachedPath(url)`, `cacheSong(url)`, `getAllCachedFiles()`, `clearCache()`, `deleteCacheByUrl()`
   - 使用 `dio` 库下载，支持下载进度回调

3. **music_player_controller.dart**
   - `playlist` 改为可变列表（`List.of(localPlaylist)`），不再用 `static const`
   - 新增 `_cacheChecked` 和 `_cachingInProgress` 字段，防止重复扫描/下载
   - `_scanCachedPaths()`：加载 playlist 时扫描所有网络歌曲的本地缓存并填充 `cachedPath`
   - `_buildAudioSource()`：优先使用本地缓存文件（`Uri.file()`），若缓存文件不存在则回退到网络 URL
   - `_cacheCurrentTrackIfNeeded()`：播放网络歌曲后异步下载到缓存，含去重保护
   - 在 `play()`, `nextTrack()`, `previousTrack()`, `selectTrack()` 中调用缓存触发方法

4. **cache_browser_page.dart**
   - 导入 `MusicCacheService`
   - 新增「音乐缓存」分组（图标：🎵 music_note），扫描 `music_cache` 目录
