## 修复：audio_service FlutterFragmentActivity 错误（最终方案）
- **错误**：`PlatformException: The Activity class declared in your AndroidManifest.xml is wrong or has not provided the correct FlutterEngine`
- **根因**：`audio_service` 的 `AudioServicePlugin.getFlutterEngine()` 只识别两种基类：`FlutterActivity` 或 `AudioServiceFragmentActivity`。继承 `FlutterFragmentActivity` 不在识别范围内，导致 `wrongEngineDetected=true` 并抛出异常。
- **最终修复**：继承 `AudioServiceFragmentActivity`（audio_service 专门提供的基类，内部已封装了 FlutterEngine 缓存注册）
  ```kotlin
  import com.ryanheise.audioservice.AudioServiceFragmentActivity
  class MainActivity : AudioServiceFragmentActivity()
  ```
- **注意**：修改原生层后必须完全重新构建（flutter run，不能只热重启）

## 修复：通知栏无播放控制 + 通知权限入口不显示
- **原因**：AndroidManifest 缺少 `POST_NOTIFICATIONS` 声明，且 main.dart 未运行时请求通知权限
- **修复1** `AndroidManifest.xml`：在 `FOREGROUND_SERVICE` 权限后添加
  ```xml
  <uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
  ```
- **修复2** `main.dart`：import permission_handler，在 JustAudioBackground.init 之前请求
  ```dart
  await Permission.notification.request();
  ```
- **说明**：Android 13+(API 33) 强制要求，permission_handler 在 Android 12 及以下会自动跳过

## 修复：通知栏有通知但无播放控制按钮（上/下一首、播放/暂停）
- **根因**：原先用 `setAudioSource(单曲)` 逐曲加载，`sequence` 只有1条，`hasNext/hasPrevious` 均为 false，导致 `_broadcastState()` 的 controls 数组只有 `[play/pause, stop]`，compact view 展示不出任何操作按钮
- **修复**：改用 `setAudioSources(完整列表)` 一次性加载所有曲目（`ConcatenatingAudioSource` 已废弃，新 API 为 `setAudioSources`）
  - `nextTrack/previousTrack` 改用 `seekToNext/seekToPrevious`
  - `selectTrack(index)` 改用 `seek(Duration.zero, index: index)`
  - 曲目切换通过 `currentIndexStream` 监听，自动加载对应歌词
- **注意**：每次换曲不再重新 setAudioSource，只 seek 到新 index

## 修复：GetX Obx 使用错误（music_player_page.dart）

- **错误**：`[Get] the improper use of a GetX has been detected` - Obx 无法追踪在 `itemBuilder` 回调内部访问的可观察变量
- **位置**：`_PlaylistBottomSheet` 类中的播放列表（第 620 行）
- **根因**：原代码将 `Obx` 包裹在 `ListView.builder` 外层，但 `controller.currentIndex.value` 是在 `itemBuilder` 回调内部访问的。GetX 只能追踪在 builder 函数**直接返回**的 widget 树中访问的可观察变量，无法追踪在回调函数中访问的变量。
- **修复**：
  - 移除 `ListView.builder` 外层的 `Obx` 包裹
  - 在 `itemBuilder` 回调函数内部，用 `return Obx(() { ... });` 包裹每个列表项
  - 这样每个列表项都在 `Obx` 的追踪范围内，`currentIndex` 变化时只有受影响的列表项会重建
- **代码变更**：`lib/pages/music/music_player_page.dart` 第 618-695 行

## 优化：MiniPlayerBar 显示条件收紧

### 需求
1. 只有 `hasStartedPlaying` 为 true（用户真实点击过播放）才显示
2. 只有在首页**音乐 Tab**（index=0）才显示浮层

### 修改内容
- **`lib/pages/music/mini_player_bar.dart`**：
  - 新增构造参数 `currentTabIndex`（由 HomePage 传入）

- **`lib/pages/home/home_page.dart`**：
  - `MiniPlayerBar()` 改为 `MiniPlayerBar(currentTabIndex: _currentIndex)`
  - 修复弃用 API：`withOpacity(0.08)` → `withValues(alpha: 0.08)`

### 效果
- 切换至短视频/消息/我的 Tab 时，MiniPlayerBar 立即消失
- 未播放过任何歌曲时，任何 Tab 都不显示浮层
