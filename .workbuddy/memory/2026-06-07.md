## chat_page reverse: true 方案

最终方案：`ListView.builder(reverse: true)` 从架构上解决路由动画抖动问题。

### 核心思路
`pixels = 0` 天然就是底部，数据任何时候到都在底部，无需 initialScrollOffset、条件渲染、postFrameCallback jumpTo 等 timing trick。

### 改动清单
1. ListView.builder 加 `reverse: true`，SizedBox 移到 `index == 0`
2. `_messages` 存储顺序改为**新→旧**（index 0 = 最新，位于底部）
3. 所有 `_messages.add(msg)` → `_messages.insert(0, msg)`
4. `_initConversation` 中 `addAll(msgs.reversed)`，删掉 postFrameCallback + jumpTo
5. `_scrollToBottom` / `didChangeMetrics` 中的 `maxScrollExtent` → `minScrollExtent`
6. `ScrollController` 恢复默认构造

## 音乐模块：增加网络歌曲缓冲中的加载状态

在播放器控制器中新增 `isBuffering` 可观察状态（监听 processingStateStream），当播放网络歌曲（如 SoundHelix-Song）需要缓冲数据时自动触发。

6 个文件改动一览：
1. **music_player_controller.dart** — 新增 `isBuffering` 变量 + processingStateStream 监听
2. **music_player_page.dart** — 全屏播放页加载遮罩增加 isBuffering 检测，显示"缓冲中..."
3. **mini_player_bar.dart** — 播放/暂停按钮在缓冲时显示加载动画，进度条显示 indeterminate 动画
4. **playlist_bottom_sheet.dart** — 当前曲目缓冲时，右侧图标替换为加载动画
5. **music_playback_controls.dart** — 新增 isBuffering 参数，缓冲时播放按钮显示加载动画，上下首按钮置灰
6. **local_ranking_item.dart** — 增加 isCurrentBuffering 检测，覆盖层显示加载动画，右侧标签显示"缓冲中"

## 第二次优化：点击网络歌曲后立即显示 MiniPlayerBar

修改 `selectTrack`、`nextTrack`、`previousTrack` 方法，在方法开头立即设置 `hasStartedPlaying = true`（保证 MiniPlayerBar 即刻出现），若目标歌曲是网络歌曲则同时设置 `isBuffering = true`，不等 `_ensurePlaylistLoaded()` 或缓冲完成，用户点击后零延迟看到底部的加载状态。
