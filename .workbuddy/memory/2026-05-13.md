  - `isDark ? Colors.white12 : Colors.grey[300]` → `colorScheme.outline.withValues(alpha: 0.3)`
  - `isDark ? Color(0xFF3A3A3C) : colorScheme.surface` → `colorScheme.surfaceContainer`

  - `isDark ? Colors.white : colorScheme.onPrimary` → `colorScheme.onPrimary`

## 新增 WebRTC 视频通话模块

### 后端（cuberverse 项目已有）
- NestJS 信令服务器，端口 3002
- 协议：WebSocket JSON（event + data）
- 事件：connected, join-room, leave-room, user-joined, user-left, room-users, offer, answer, ice-candidate, end-call, call-ended

### Flutter 端新增文件
| 文件 | 说明 |
|------|------|
| `lib/webrtc/models/call_state.dart` | 通话状态枚举（idle/calling/incoming/connecting/connected/ended/error） |
| `lib/webrtc/models/signaling_message.dart` | 信令消息模型（base + 各事件类型） |
| `lib/webrtc/services/signaling_client.dart` | WebSocket 信令客户端（GetxService，全局单例） |
| `lib/webrtc/services/webrtc_controller.dart` | WebRTC 核心控制器（GetxController，管理 PC/媒体流/信令协调/通话状态） |
| `lib/webrtc/pages/call_entry_page.dart` | 入口页：输入服务器地址/房间号/昵称 |
| `lib/webrtc/pages/video_call_page.dart` | 视频通话页：远端主画面 + 本地画中画 + 底部控制栏（麦克风/摄像头/翻转/挂断）|

### 修改的文件
- `pubspec.yaml`：添加 `flutter_webrtc: ^0.12.0` 依赖
- `lib/main.dart`：注册 `SignalingClient` 全局单例
- `lib/routes/app_routes.dart`：添加 `/webrtc-call` 路由

### 架构设计
- 每个通话生命周期创建新的 `WebrtcController`（`Get.put(permanent:true)` 后 `Get.delete` 清理）
- `SignalingClient` 全局单例，WebSocket 连接由控制器的 `startCall()` 触发，`endCall()` 时断开
- STUN 服务器使用 Google 公共 STUN（stun:stun.l.google.com:19302）
- UI 使用 `Obx` 响应式更新通话状态，PopScope 防止误触返回
